# Handoff bundle — Testing & Product

Three files, shared **2026-06-15**, for the testing and product teams of the Police Academy Admissions Platform.

| File | For | What it is |
|---|---|---|
| [TESTING_GUIDE.md](TESTING_GUIDE.md) | Testing / QA | Environments + URLs, test accounts/NIDs, expected-behavior sources, negative-test catalog, bug-report template |
| [PRODUCT_GUIDE.md](PRODUCT_GUIDE.md) | Product | Spec → current-state → gaps map, personas, story-writing checklist |
| [police_academy_detailed_brd_scope_md.md](police_academy_detailed_brd_scope_md.md) | Both | The BRD — the requirement source both guides point back to |
| [CLAUDE.md](CLAUDE.md) | Reference | Full engineering operating context — architecture, all routes (§4), RBAC roles (§5), and the running done-log (§11) of what's built |

> **Note:** The two guides are *routers* — they reference additional repo docs (e.g. the eligibility test matrix, DB constraints, CLAUDE.md). Links to the BRD (in this bundle) resolve; links to other repo docs require repo access. Ask for the full repo, or for standalone versions with that data inlined, if needed.
